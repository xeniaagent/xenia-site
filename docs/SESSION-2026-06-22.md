## Details — what each area changed

### 1. SF Neighborhood Map (the big addition)

**Before this session**: no map.

**After this session**: an interactive SVG map sits in the bottom-right of the hero, anchored to the hero section, hidden on mobile.

- **41 SF Planning polygons** drawn from the official `data.sfgov.org` GeoJSON
- **27 MLS-data neighborhoods** get labels (polylabel algorithm + force-based repulsion to prevent overlap)
- **14 non-MLS neighborhoods** render in the same gold color but have no label and aren't clickable
- **Hover** any label: that label grows from 13px to 26px, the polygon's stroke thickens to 6px, both turn bright orange (`#ff8a3d`)
- **Click** any label: opens the existing neighborhood detail panel, scrolls to it smoothly
- **Wrap opacity**: 60% baseline, never changes on hover
- **Mobile**: hidden via `@media (max-width: 880px) { display: none }`

Key implementation details:
- SVG inlined into the HTML (in a `<script type="text/plain">` block) so it works in `file://` contexts
- Douglas-Peucker simplification with 0.00003° tolerance (≈3m) so coastlines don't collapse
- `setAttribute('style', fullString)` pattern for hover styles because `el.style.X = 'Y'` is unreliable on innerHTML-injected SVG in some browsers

### 2. Chart slider

The neighborhood panel's 24-month price chart now has a slider under it.

- **Slider** sits below the SVG, padded to match the SVG's `PAD_L/PAD_R` (11.67% / 4%) so the thumb visually aligns with the dashed indicator line on the chart at every month
- **Readout** (e.g. "Jun 2026 · $1.8M") lives in the **chart heading row, right side** — not below the chart where it competed with the source line for attention
- **Metric toggle** (Sale / $/sqft / DOM) re-renders the chart and resets the slider to the most-recent month
- **Keyboard**: arrow keys step ±1 month, Home/End jump to start/end, focus-visible ring on the thumb

### 5. Contact form fix (the silent lead-drop bug)

**Before**: the form had `onsubmit="event.preventDefault(); this.querySelector('.thanks').hidden = false; this.reset();"` — meaning the form **showed a "Thanks!" but never sent anything**. Every contact message was silently lost.

**After (interim)**: replaced with a `mailto:` form submission. When the visitor hits Send, their email client opens with a pre-filled message to `xenia.brown@vanguardproperties.com` containing the form contents.

**Still pending (2026-06-23 todo)**: replace `mailto:` with a real backend via Formspree. Requires you to sign up at `formspree.io` (~2 min) and paste me the endpoint URL.
